# DX혁신실 회식 추첨 시스템 — Claude Code 작업 가이드

이 파일은 Claude Code가 작업할 때 자동으로 읽는 프로젝트 컨텍스트입니다.

## 프로젝트 개요

DX혁신실 격주 회식 멤버 추첨 웹앱. 실장님 화합 다지기 지시로 시작.
단일 페이지 정적 웹앱(서버 없음, localStorage로 영구 저장).

**서울 근무자 제외 후 25명** (실장·휴가자 제외 + TF는 PMO 3명만 잔류).

> ⚠ **현재 로스터 26명** (2026-06-10 허성오 기획 T5 추가) — 연간 플랜 전제(25명·144자리)가
> 깨져서 `seasonPlanValid()`가 false → **시즌 엔진 비활성, 레거시 추첨으로 동작 중**.
> 시즌을 살리려면 26명 기준 쿼터 재설계(또는 로스터 원복)가 필요. 호연의 결정 대기.

## 핵심 비즈니스 룰

### 컬럼(축) 구조 — `data.js`의 `COLUMNS`
추첨 UI는 4개 세로 컬럼. 멤버의 실제 `group`(기획·운영·정보보호·PMO)과 분리됨:
1. **기획** — group=기획에서 1명
2. **운영** — group=운영에서 1명
3. **정보보호+PMO** — group ∈ {정보보호, PMO}에서 1명 (`groups: ['정보보호','PMO']`)
4. **잔여 통합** (`leftover: true`) — 앞 3컬럼에서 안 뽑힌 전원에서 1명 (네온 그린 표시)

### 회차 구조
- 1회차 = 3팀 × 4인 = 12명 (단 PEER 하위팀은 아래 예외)
- 매월 2주차 **MIX 모드**: 각 팀 4명이 모두 다른 티어 (T1~T6 중 4개)
- 매월 4주차 **PEER 모드**: 한 팀 전체가 같은 묶음 (상=T1+T2+T3 / 중=T4+T5 / 하=T6)
  - **PEER 하위(T6) 예외**: 정보보호+PMO에 T6가 없어 컬럼 구조 포기.
    T6 전원에서 올랜덤 4명 추첨 (`POOLS` 하위에 `allRandom:true`).
    UI에 🎲 시안 플래그 + `slotCol:null, allRandom:true`로 표시.

### 연간 플랜(시즌) — 2026 하반기 균등화
`data.js`의 `SEASON` 상수 + `app.js`의 시즌 엔진 섹션. 7~12월 매월 MIX+PEER = 12회차.
- **쿼터**: T2 = 5회, 나머지 19명 = 6회 (합 144 = 월 24자리 × 6개월). max−min=1이 수학적 최선.
- **월간 중복 금지**: 같은 달 1회만 출석 → 비T2는 만근 강제, **매월 "미휴식 T2" 1명이 휴식**
  (`pickSeasonRester`, 휴식자는 그 달 양쪽 회차 레코드에 `restedName`으로 저장).
- **MIX 구성 강제**: 상위(T1~3) 7 / 중위(T4~5) 4 / T6 1 — `drawMixSeason`이 밴드 쿼터를
  백트래킹에 결합 (`arrangeMixSeason`). "선뽑고 후배치"는 61~63% 실패하므로 분리 금지.
- **PEER = 여집합**: 같은 달 MIX가 있으면 멤버십 완전 강제, 컬럼은 표시용 배정만
  (`assignPeerColumns`, 그룹 불일치 멤버는 `offColumn` 플래그 → 그린 "교차 배치" 표시).
- **PEER 먼저 뽑는 경우**: 티어 제약 + 여집합 실배치 검사 통과까지 재선발 (`selectPeerBands`).
- **페어링은 `seasonMonth` 저장 필드 기준** — roundNum 파생 계산 절대 금지 (삭제 시 붕괴).
- **로스터 변경 감지**: `seasonPlanValid()`가 false면 레거시 추첨 폴백 + 경고 토스트.
  시즌 중 인사 변경은 쿼터 설계 전체 재산정 필요 — 호연에게 알릴 것.
- 시즌 회차 레코드: `{ season: '2026H2', seasonMonth: 1~6, restedName }` 필드 추가 (하위호환).

### 쿨다운
- **시즌 중**: 소프트만 (`seasonSoftWeights` — 직전 같은 모드 회차 출석 시 가중치 페널티).
  하드 쿨다운은 쿼터와 수학적으로 모순 (T1은 MIX 상위 7/11 구조상 연속 출석 불가피).
- **시즌 외(레거시)**: 한 번 뽑힌 사람은 다음 **같은 모드** 회차에서 자동 제외 (1회차 쿨다운)
- MIX와 PEER는 별개 쿨다운 풀

### 잔여 통합 컬럼 = 옛 fallback 대체
- 옛 버전의 "정보보호 가상슬롯→TF 재추첨(인원조정)" 시스템은 **제거됨**.
- 이제 빠듯한 컬럼(정보보호+PMO 등)의 부족분은 잔여 통합 컬럼이 자연스럽게 흡수.

## 티어 구조 (25명)

| 티어 | 정의 | 기획 | 운영 | 정보보호 | PMO |
|---|---|---|---|---|---|
| T1 그룹장 | 권재범, 황지영, 남현태 | 1 | 1 | 1 | 0 |
| T2 팀장급 | 직책자 | 2 | 2 | 1 | 1(이거성) |
| T3 시니어급 | P5 미직책 + 고연차 E직군 | 1 | 1 | 1 | 0 |
| T4 차장급 | P4 + 한희정(E4) | 4 | 0 | 1 | 1(정경화) |
| T5 과장급 | P3 과장 | 0 | 1 | 0 | 1(부병식) |
| T6 주니어급 | 사원·대리·주임 | 1 | 4 | 0 | 0 |

특이사항:
- **실장 신대호 상무 제외**, **박현규 대리 휴가로 제외**
- **이선규 등 TF 비PMO 전원 서울 근무로 제외** — TF 잔류는 PMO 3명(이거성·정경화·부병식)뿐, group='PMO'로 재배치
- T6는 기획·운영에만 존재 → PEER 하위가 운영 쏠림 (그래서 올랜덤 처리)
- 김호연(본인)은 기획 T6 단인 슬롯
- 박영선·강순옥은 직급 아닌 연차 기준으로 T3 배치

## 파일 구조 및 책임

| 파일 | 역할 | 수정 빈도 |
|---|---|---|
| `index.html` | HTML 구조만, 거의 안 건드림 | 낮음 |
| `styles.css` | 디자인 변경 시 | 중간 |
| `data.js` | 인사 변경(이동/입사/퇴사/휴가) 시 | 자주 |
| `app.js` | 알고리즘·UI 로직 변경 시 | 자주 |
| `history-seed.js` | 공유 이력 시드 (이력 탭 내보내기로 생성, 직접 수정 금지) | 이력 공유 시 |

### 이력 공유 (DB 없음)
localStorage는 브라우저별이라 GitHub Pages URL을 공유해도 이력은 공유 안 됨.
- 이력 탭 **"공유 파일 내보내기"** → `history-seed.js` 다운로드 → 저장소에 덮어쓰고 push
- 로드 시 `mergeSeedHistory()`가 시드와 로컬 이력을 병합:
  동일성 키는 `roundKey()` (시즌 회차 = 시즌·월·모드 / 일반 회차 = 확정 시 부여된 `id`)
- 키 충돌 시 로컬 우선, 삭제한 시드 회차는 `state.deletedSeedKeys`(묘비)로 복원 차단
- 회차 레코드에 `id` 필드 추가됨 (기존 레코드는 로드 시 자동 부여 — 하위호환)

### 이력 자동 복구
`repairHistory()`가 로드 시 정원(4명) 미달 팀을 감지해 규칙에 맞는 후보를 랜덤 보충:
회차 내 중복 금지, 시즌이면 휴식자·같은 달 출석자 제외 + MIX 밴드 쿼터 유지,
MIX는 팀 내 티어 상이 + 빈 컬럼 그룹 엄수, PEER는 같은 묶음 티어(하위는 올랜덤 형식 유지),
앞뒤 같은 모드 회차 출석자는 소프트 회피. 보충 내역은 토스트로 표시.

## 자주 일어나는 작업 패턴

### 1. 인원 변경
- `data.js`의 `MEMBERS` 배열만 수정
- 새 멤버 객체 형식: `{ name, group, tier, title, part }`
- 그룹: `기획 | 운영 | 정보보호 | PMO`
- 티어: `T1 | T2 | T3 | T4 | T5 | T6`
- 컬럼 구성 변경은 `COLUMNS` 배열 (leftover 플래그 = 잔여 통합 컬럼)

### 2. 알고리즘 튜닝
- `app.js`의 `drawDifferentMode()` (MIX) 또는 `drawSimilarMode()` (PEER)
- 컬럼 후보 산출은 `columnCandidates(col, ...)` 공통 함수
- MIX는 `backtrackDifferent()` 백트래킹 (leftover 컬럼을 항상 마지막에 처리)
- PEER 하위 올랜덤은 `drawSimilarTeam()` 안 `pool.allRandom` 분기
- 쿨다운 길이는 `getCooldown()` — 현재 1회차

### 3. UI 튜닝
- 슬롯머신 애니메이션: `animateDraw()` (`spinDuration`, `rowDelay` 변수)
- 색상/테마: `styles.css` 상단 CSS 변수 (`--neon-pink`, `--neon-cyan` 등)
- 잔여 통합 락 = `.drum.locked-leftover` (그린), 올랜덤 락 = `.drum.locked-random` (시안)
- PEER 테마는 `body.theme-peer` 셀렉터로 오버라이드

## 작업 시 주의사항

- **확정된 이력은 localStorage에 저장됨** — 코드 수정으로 인해 데이터 깨질 가능성 있을 때 호연한테 미리 알릴 것
- **STORAGE_KEY는 변경 시 기존 이력 사라짐** — 버전 올릴 때 신중히
- `data.js`의 멤버 순서/구조 변경 시 알고리즘이 영향받지 않는지 확인
- 새 기능 추가 시 README.md의 "기능" 섹션도 업데이트
- 한국어 주석 OK, 변수명은 영문
