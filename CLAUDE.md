# DX혁신실 회식 추첨 시스템 — Claude Code 작업 가이드

이 파일은 Claude Code가 작업할 때 자동으로 읽는 프로젝트 컨텍스트입니다.

## 프로젝트 개요

DX혁신실(4개 그룹·38명)의 격주 회식 멤버 추첨 웹앱.
실장님 화합 다지기 지시로 시작. 단일 페이지 정적 웹앱(서버 없음, localStorage로 영구 저장).

## 핵심 비즈니스 룰

### 회차 구조
- 1회차 = 3팀 × 4인 = 12명
- 매월 2주차 **MIX 모드**: 각 팀 4명이 모두 다른 티어 (T1~T6 중 4개)
- 매월 4주차 **PEER 모드**: 상/중/하 묶음에서 각 1팀씩
  - 상위 묶음: T1+T2+T3
  - 중위 묶음: T4+T5
  - 하위 묶음: T6
- 한 팀 = 4그룹(TF·기획·운영·정보보호)에서 1명씩

### 쿨다운
- 한 번 뽑힌 사람은 다음 **같은 모드** 회차에서 자동 제외 (1회차 쿨다운)
- MIX와 PEER는 별개 쿨다운 풀

### 자동 보충 (Fallback)
- 쿨다운 또는 인원 부족으로 슬롯이 비면:
  - 같은 모드 제약(다른 티어 또는 같은 묶음) 유지하면서
  - **다른 그룹**(인원 많은 그룹 우선)에서 1명 더 뽑음
- 결과: 그 팀에 같은 그룹 2명이 들어가게 됨
- UI에서 보충 인원은 보라색 글로우 + "OO 보충" 배지

## 티어 구조 (38명)

| 티어 | 정의 | TF | 기획 | 운영 | 보호 |
|---|---|---|---|---|---|
| T1 그룹장 | 이선규, 권재범, 황지영, 남현태 | 1 | 1 | 1 | 1 |
| T2 팀장급 | 직책자 (리더 + 팀장 부장/차장) | 6 | 2 | 2 | 1 |
| T3 시니어급 | P5 미직책 + 고연차 E직군 | 0 | 1 | 1 | 1 |
| T4 차장급 | P4 + 한희정(E4 '94 입사) | 2 | 4 | 0 | 1 |
| T5 과장급 | P3 과장 | 3 | 0 | 1 | 0 |
| T6 주니어급 | 사원·대리·주임 | 4 | 1 | 4 | 0 |

특이사항:
- **실장 신대호 상무 제외** (어느 그룹에도 안 들어감)
- **박현규 대리 휴가로 제외** (정보보호 T6은 그래서 0명)
- 정보보호는 인원이 적어 fallback 대상으로 자주 보충됨
- 김호연(본인)은 기획 T6 단인 슬롯
- 박영선·강순옥은 직급(차장·과장) 아닌 연차 기준으로 T3에 배치

## 파일 구조 및 책임

| 파일 | 역할 | 수정 빈도 |
|---|---|---|
| `index.html` | HTML 구조만, 거의 안 건드림 | 낮음 |
| `styles.css` | 디자인 변경 시 | 중간 |
| `data.js` | 인사 변경(이동/입사/퇴사/휴가) 시 | 자주 |
| `app.js` | 알고리즘·UI 로직 변경 시 | 자주 |

## 자주 일어나는 작업 패턴

### 1. 인원 변경
- `data.js`의 `MEMBERS` 배열만 수정
- 새 멤버 객체 형식: `{ name, group, tier, title, part }`
- 그룹: `TF | 기획 | 운영 | 정보보호`
- 티어: `T1 | T2 | T3 | T4 | T5 | T6`

### 2. 알고리즘 튜닝
- `app.js`의 `drawDifferentMode()` (MIX) 또는 `drawSimilarMode()` (PEER)
- 쿨다운 길이는 `getCooldown()` — 현재 1회차
- 보충 우선순위는 백트래킹 안에서 "인원 많은 그룹 우선" 정렬

### 3. UI 튜닝
- 슬롯머신 애니메이션: `animateDraw()` (`spinDuration`, `rowDelay` 변수)
- 색상/테마: `styles.css` 상단 CSS 변수 (`--neon-pink`, `--neon-cyan` 등)
- PEER 테마는 `body.theme-peer` 셀렉터로 오버라이드

## 작업 시 주의사항

- **확정된 이력은 localStorage에 저장됨** — 코드 수정으로 인해 데이터 깨질 가능성 있을 때 호연한테 미리 알릴 것
- **STORAGE_KEY는 변경 시 기존 이력 사라짐** — 버전 올릴 때 신중히
- `data.js`의 멤버 순서/구조 변경 시 알고리즘이 영향받지 않는지 확인
- 새 기능 추가 시 README.md의 "기능" 섹션도 업데이트
- 한국어 주석 OK, 변수명은 영문
