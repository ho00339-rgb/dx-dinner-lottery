# DX혁신실 회식 추첨 시스템

실장님 지시에 따른 DX혁신실 4개 그룹 간 화합 다지기용 격주 회식 추첨 웹앱.

## 실행 방법

`index.html` 더블클릭 → 브라우저에서 열림. 끝.
서버 같은 거 필요 없음. 회사 PC 어디서나 작동.

## 컬럼(축) 구조

서울 근무자 제외 후, 추첨은 4개 세로 컬럼으로 진행:

1. **DX 기획** — 기획 그룹에서 1명
2. **DX 운영** — 운영 그룹에서 1명
3. **정보보호 + PMO** — 정보보호 그룹 + (TF 잔류) PMO 3명에서 1명
4. **잔여 통합** — 위 3개 컬럼에서 안 뽑힌 사람을 모두 합쳐 1명 (네온 그린으로 표시)

## 기능

- **MIX 모드** (매월 2주차): 한 팀 4명이 모두 다른 티어, 3팀 × 4인 = 12명
- **PEER 모드** (매월 4주차): 한 팀 전체가 같은 묶음(상/중/하)
  - 단, **하위(T6)는 정보보호+PMO에 T6가 없어** 컬럼 구조를 포기하고 T6 전원에서 올랜덤 추첨 (🎲 시안 표시)
- **잔여 통합 컬럼**: 앞 3개 컬럼에서 뽑히지 않은 인원을 합쳐 추첨 (티어 규칙은 유지)
- **쿨다운 1회차**: 한 번 뽑힌 사람은 다음 같은 모드 회차에서 자동 제외
- **이력 저장**: 확정한 회차는 브라우저에 영구 저장 (이력 탭에서 확인)
- **회차 삭제**: 이력 탭에서 한 회차씩 삭제 (확인 팝업 후 삭제, 남은 회차 번호 자동 재정렬)
- **출장 카운트**: 멤버 탭에서 누가 몇 번 나갔는지 한눈에

## 파일 구조

| 파일 | 역할 |
|---|---|
| `index.html` | HTML 골격, 화면에 보일 요소들 |
| `styles.css` | 모든 디자인/색상/애니메이션 |
| `data.js` | 인사 정보 (멤버/그룹/티어) — **인원 변경 시 여기만 수정** |
| `app.js` | 추첨 로직, 슬롯머신, 이력 관리 |
| `CLAUDE.md` | Claude Code용 프로젝트 컨텍스트 |

## 인원 변경 시

`data.js` 열어서 `MEMBERS` 배열만 수정. 다른 파일 건드릴 필요 없음.

예시:
```javascript
// 새 멤버 추가 (group: 기획 | 운영 | 정보보호 | PMO)
{ name: '홍길동', group: '기획', tier: 'T4', title: '차장', part: '과제발굴' },

// 빠진 멤버는 해당 줄 삭제
```

> 컬럼 구성을 바꾸려면 `data.js`의 `COLUMNS` 배열 수정.

## 티어 정의

| 티어 | 의미 | PEER 묶음 |
|---|---|---|
| T1 | 그룹장 (권재범/황지영/남현태) | 상위 |
| T2 | 팀장급 (직책자) | 상위 |
| T3 | 시니어급 (P5 미직책자 + 고연차 E직군) | 상위 |
| T4 | 차장급 (P4 + 한희정 E4) | 중위 |
| T5 | 과장급 (P3) | 중위 |
| T6 | 주니어급 (P2/P1/E3) | 하위 (올랜덤) |

## 운영 룰

- 서울 근무자 제외: TF는 PMO 3명(이거성·정경화·부병식)만 잔류, 나머지 전원 제외 (실장·휴가자 제외 포함 25명)
- MIX 1회차당 3팀 × 4인 = 12명, 격주 진행 (매월 2주차 MIX / 4주차 PEER)
- 정보보호+PMO처럼 빠듯한 컬럼은 잔여 통합 컬럼이 부족분을 흡수
- 호연(기획 T6), 김혁중(운영 T5), 권태근(보안 T4) 등은 단인 슬롯이라 쿨다운으로 격월 출장
